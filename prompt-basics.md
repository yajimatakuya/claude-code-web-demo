# プロンプトの基礎

## はじめに

プロンプトとは、AI（人工知能）モデルに対して与える指示や質問のことです。効果的なプロンプトを作成することで、AIから望ましい回答や結果を得ることができます。本記事では、プロンプトの基本的な概念から実践的なテクニックまでを解説します。

## プロンプトとは

プロンプトは、AIモデルとのコミュニケーションの入り口です。適切なプロンプトを設計することで、AIの能力を最大限に引き出すことができます。

### プロンプトの構成要素

1. **指示（Instruction）**: AIに何をしてほしいかを明確に伝える
2. **コンテキスト（Context）**: 背景情報や前提条件を提供する
3. **入力データ（Input）**: 処理してほしいデータや情報
4. **出力形式（Output Format）**: 期待する回答の形式やスタイル

## 基本的なプロンプト技法

### 1. 明確で具体的な指示

曖昧な指示ではなく、明確で具体的な指示を与えることが重要です。

**悪い例:**
```
文章を書いて
```

**良い例:**
```
プログラミング初心者向けに、Pythonの変数について300文字程度で説明する文章を書いてください。
専門用語は避け、日常的な例を使って説明してください。
```

### 2. コンテキストの提供

AIに十分な背景情報を与えることで、より適切な回答を得られます。

**例:**
```
私は高校の数学教師です。三角関数を初めて学ぶ生徒向けに、
日常生活での三角関数の応用例を3つ挙げて、それぞれ簡単に説明してください。
```

### 3. ステップバイステップの指示

複雑なタスクは、段階的な指示に分解することで、より正確な結果が得られます。

**例:**
```
以下の手順でコードレビューを行ってください：
1. まず、コードの全体的な構造を分析してください
2. 次に、潜在的なバグや問題点を指摘してください
3. 最後に、改善提案を3つ挙げてください
```

### 4. 役割の設定（ロールプレイ）

AIに特定の役割を与えることで、その視点からの回答を得られます。

**例:**
```
あなたは経験豊富なソフトウェアアーキテクトです。
マイクロサービスアーキテクチャのメリットとデメリットについて、
実務的な観点から説明してください。
```

### 5. 例示（Few-shot Learning）

期待する出力の例を示すことで、AIに形式やスタイルを学習させます。

**例:**
```
以下の形式で、3つの都市について説明してください：

例: 東京 - 日本の首都で、世界最大級の都市圏を形成する。
伝統と最先端技術が共存する国際都市。

都市: パリ、ニューヨーク、シンガポール
```

## プロンプトの最適化テクニック

### 明確な制約を設定する

- 文字数や単語数の制限
- 使用する言語やトーン
- 含めるべき要素、避けるべき要素

**例:**
```
500文字以内で、フォーマルな文体を使用し、
専門用語は避けて、機械学習の基本概念を説明してください。
```

### 出力形式の指定

期待する出力の形式を明確に指定します。

**例:**
```
以下の情報をJSON形式で出力してください：
- 名前
- 年齢
- 職業
- 趣味（配列）
```

### 反復的な改善

最初のプロンプトが完璧でなくても、結果を見ながら段階的に改善していきます。

1. 基本的なプロンプトを試す
2. 結果を評価する
3. 不足している情報や指示を追加する
4. 再度試して改善する

## よくある間違いと対処法

### 1. 曖昧な指示

**問題:** 「良い文章を書いて」
**解決策:** 「ビジネスメール形式で、丁寧な言葉遣いを使用し、300文字程度の挨拶文を書いてください」

### 2. コンテキスト不足

**問題:** 「これについて説明して」
**解決策:** 何について説明してほしいのか、誰向けなのかを明確に指定する

### 3. 複雑すぎる指示

**問題:** 一度に多くのタスクを詰め込む
**解決策:** タスクを分解し、段階的に進める

### 4. 出力形式の未指定

**問題:** どのような形式で回答してほしいか不明確
**解決策:** 箇条書き、表、JSON、コードブロックなど、具体的に指定する

## 実践例

### 例1: データ分析の依頼

```
以下のCSVデータを分析してください：

売上データ: [データ]

以下の項目について分析結果を報告してください：
1. 月別の売上推移
2. 最も売れている商品トップ3
3. 売上が減少している時期とその考えられる原因

結果は箇条書きで、各項目に簡単な説明を添えてください。
```

### 例2: コード生成の依頼

```
Pythonで、以下の機能を持つ関数を作成してください：

機能: リストの中から重複を除去し、ソートして返す
入力: 整数のリスト
出力: 重複を除去しソートされたリスト

要件:
- 関数名は remove_duplicates_and_sort とする
- docstringを含める
- 型ヒントを使用する
- エラーハンドリングを含める
```

### 例3: 文章の要約

```
以下の記事を200文字以内で要約してください。
要約には以下の要素を含めてください：
- 主要なポイント
- 重要な結論
- 実務への応用可能性

記事: [記事テキスト]
```

## プロンプトエンジニアリングのベストプラクティス

1. **明確性**: 曖昧さを避け、具体的に指示する
2. **構造化**: 情報を整理して提示する
3. **コンテキスト**: 十分な背景情報を提供する
4. **反復**: 結果を見ながら改善する
5. **実験**: 異なるアプローチを試す
6. **検証**: 出力結果を必ず確認する

## まとめ

効果的なプロンプトを作成するには、以下のポイントが重要です：

- AIに何をしてほしいかを明確に伝える
- 十分なコンテキストと制約を提供する
- 期待する出力形式を具体的に指定する
- 複雑なタスクは段階的に分解する
- 例を示して期待する結果を明確にする

プロンプトエンジニアリングは、実践を通じて上達していくスキルです。様々なプロンプトを試し、結果を観察しながら、徐々に改善していくことが大切です。
